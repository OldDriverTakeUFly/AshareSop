# A 股盘后总结 Skill

每天收盘后生成结构化的盘后总结：板块涨幅 → 领涨个股 → 热点方向 → 催化原因 → 总结结论。

## 定位

**daily-market-scan 的下游消费者。** 读取其已采集到 SQLite 的盘面数据（涨停/龙虎榜/资金流），结合 web 搜索找出领涨催化原因，生成 markdown 报告。不自己采集数据，不做买卖建议。

## 使用方法

```text
# 触发方式——对 agent 说：
"做今天的盘后总结"
"今天复盘一下"
"今日热点分析"
"after hours review"
```

**前置条件**：必须先运行 `daily-market-scan` skill 采集当日数据。若数据未采集，agent 会提示你先跑。

## 输出

- 目录：`docs/盘后总结/`
- 文件名：`{YYYY-MM-DD}_盘后总结.md`
- 结构：市场概览 → 板块涨幅排名 → 热点方向深度（含催化原因）→ 连板梯队 → 游资动向 → 总结结论

## 报告 6 节结构

| 节 | 内容 | 数据来源 |
|----|------|----------|
| 一、市场概览 | 涨停/炸板/跌停数 + 大盘资金流 | SQLite fund_flow_market + limit_up |
| 二、板块涨幅排名 | Top 10 板块（涨幅+主力净流入+涨停数） | SQLite fund_flow_sector |
| 三、热点方向深度 | 2-3 个核心热点（领涨个股表+催化原因） | SQLite + **web 搜索** |
| 四、连板梯队 | 各连板层级个股 | SQLite limit_up_analysis |
| 五、游资动向 | 龙虎榜亮点 | SQLite dragon_tiger_detail |
| 六、总结结论 | 今日特征 + 明日关注 | AI 归纳 |

## 催化原因的搜索策略

agent 对每个热点方向搜 2-3 次 web：
- 板块级：`"{板块名} 涨停 原因 {date}"`
- 个股级：`"{领涨个股名} 利好 消息 {date}"`
- 政策级：`"{热点主题} 政策 催化 2026"`

**诚实原则**：搜不到的标注"未找到明确催化"，不编造。每条催化附 web 来源 URL。

## 与其他 skill 的关系

| Skill | 关系 |
|-------|------|
| `daily-market-scan` | **上游**——采集数据到 SQLite |
| `invest-sop-pre-market` | 不重叠——盘前报告读 invest_* 表，本 skill 读 daily_data 表 |
| `research-report` | 不重叠——研报是深度分析，盘后总结是当日速览 |

## 局限性

- **依赖上游数据**——daily-market-scan 未跑则无法生成
- **催化搜索有时效性**——web 搜索可能遗漏非公开或极新的消息
- **不做买卖建议**——只呈现数据和分析
- **每日产物**——不更新 docs/README.md 索引，不 commit/push（由用户决定）
