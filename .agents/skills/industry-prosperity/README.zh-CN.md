# 景气度投资 Skill 说明

这份文档是 `.agents/skills/industry-prosperity/` 的中文说明版，面向人类读者。它的目标不是替代 `SKILL.md`，而是帮助团队快速理解这套 skill 想解决什么问题、适合什么时候使用，以及如何把它部署到一次具体的行业比较或个股景气度分析任务里。

真正的规范性内容仍以 `SKILL.md` 为准。如果 agent 需要执行景气度投资相关工作，应优先读取 `SKILL.md`。本 README 仅是人类视角的概览，不构成规范性来源。

## 这套 skill 解决什么问题

景气度投资是 A 股市场长期最有效的投资方法论之一，但实际应用中存在大量痛点：

- 只看绝对增速（G），忽视增速的边际变化（ΔG），在"增速高但减速"的阶段踩入估值收缩陷阱
- 2022 年新能源板块的"景气投资失效"教训：增速仍在 50% 以上但股价大跌，原因是 ΔG 转负而非景气消失
- 用单一财务指标衡量景气度，缺乏多维度交叉验证
- 无法判断一个行业或个股处于景气周期的哪个位置（加速期、减速期、还是拐点）
- 有评分引擎但不知道怎么用，或者自己重新造轮子
- 混淆行业景气（β）与个股表现（α），以为行业景气就等于每只个股都在加速

这套 skill 的核心目标，是让 agent 在处理景气度投资分析时遵守同一套最小原则：

1. ΔG 比 G 更重要，增速的边际变化是核心定价锚点
2. 六维指标交叉验证，单一指标的信号不可靠
3. 山峰理论和成长股投资时钟提供标准化的周期位置分类
4. 复用 davis_analyzer 引擎，不重写评分逻辑
5. 如实分类周期位置，不因为"行业景气"就强行说个股在加速

## 何时使用

当遇到以下任务时，适合使用这套 skill：

- 判断一个行业或个股当前处于景气周期的哪个阶段（加速、减速、拐点）
- 对成长股做周期定位，决定持有、减仓还是加仓
- 构建行业景气度排名，筛选"二次点火"标的（G 和 ΔG 同时为正）
- 验证一个"高增长"标的到底是真景气机会还是减速陷阱
- 对周期股、成长股、困境反转标的做景气度层面的定价
- 追踪分析师景气预期的边际变化（景气预期框架）

不适合使用的场景：

- 市场完全由流动性驱动（如 2014 年"水牛"行情），景气信号定价能力接近零
- 标的处于零收入阶段，没有至少 2 个季度的可比增长数据
- 需要回答"大盘涨跌"或"何时买卖"的问题，景气度框架回答的是"买什么行业"
- 标的仍在亏损，应该先用 `valuation-loss-making-targets` skill 做估值

## 如何部署

把这套 skill 部署到一次具体的景气度分析任务，建议按以下顺序操作：

1. 先读 `SKILL.md`，理解 G+ΔG 框架、六维指标体系、山峰理论、成长股投资时钟和位置感框架
2. 确认 `davis_analyzer` 已安装且 Tushare token 已配置（见 SKILL.md 第 2 节 Prerequisites）
3. 获取标的至少 4 个季度的财务数据（营收和利润的 YoY 增速）
4. 调用 `davis_analyzer.prosperity.calculate_prosperity_score` 计算景气度复合评分和 ΔG
5. 调用 `davis_analyzer.prosperity_sector.classify_stock_stage` 做周期位置分类
6. 如需行业对比，调用 `prosperity_sector.aggregate_industry_prosperity` 做行业聚合
7. 如需筛选"二次点火"标的，调用 `prosperity_sector.screen_g_delta_g_ignition`
8. 调用 `prosperity_inflection.analyze_inflection` 生成拐点分析和叙述
9. 手动收集六维指标数据（BB Ratio、稼动率、交期、库存、排产、LTA），补充引擎无法覆盖的产业中观数据
10. 用山峰理论和成长股投资时钟对周期位置做最终定性确认

部署时要注意：

- 引擎计算用的是财务数据驱动的评分和分类，六维指标需要手动收集
- 不要修改 `prosperity.py`、`prosperity_inflection.py` 或 `prosperity_sector.py` 的源代码
- ΔG 需要至少 2 个季度的数据才能计算，4 个季度以上才能用 3 季度移动平均
- 30% 是关键阈值：净利润增速降至 30% 以下时超额收益大概率下滑

## 核心框架摘要

完整框架以 `SKILL.md` 为准，这里只做概述：

### G+ΔG 框架

G 是增速的绝对值，ΔG 是增速的边际变化。市场定价的核心不是"增速有多高"，而是"增速是否还在加速"。实证排序：Δ利润增速 > Δ营收增速 > ΔROE > 营收增速 > 利润增速。边际变化类指标（Δ）的一致性强于绝对水平类指标。

30% 阈值效应：净利润增速降至 30% 以下时，超额收益显著下滑；降幅超过 50% 时进一步恶化；降速越快股价越差。

### 景气度六维指标体系

| 维度 | 指标 | 景气信号 |
|------|------|---------|
| 订单 | BB Ratio（订单出货比） | BB>1 景气上行，>1.3 超级周期 |
| 产能 | 稼动率 | >90% 涨价压力，<70% 产能过剩 |
| 物流 | 交期 | 正常 8-10 周，>16 周短缺，>30 周严重短缺 |
| 渠道 | 经销商库存 | 3-6 月正常，1-1.5 月紧张，<1 月极度短缺 |
| 需求 | 排产周期 | 12-18 月强景气，>24 月需求确定性极强 |
| 客户 | LTA（长期协议） | LTA 扩散 = 短缺被产业资本确认 |

六个维度存在正反馈循环，同向上行时景气持续性最强。交期是最前瞻的指标：交期开始缩短是景气见顶的早期信号。

### 山峰理论（开源证券）

将成长周期比作一座山。左山坡（Δg>0）是戴维斯双击机会区，增速在加速，EPS 和 PE 双双上行。山顶是奇点时刻（G 最高，Δg 趋零）。右山坡（Δg<0 但 G 仍正）是估值收缩风险区，EPS 上行但 PE 收缩。山后（G 转负）是戴维斯双杀区。

投资规则：买在左山坡（加速期），卖在右山坡起点（减速刚开始时）。

### 成长股投资时钟（华泰证券）

用速度（G）和加速度（ΔG）两个维度划分四个象限：奇点（渗透率突破临界点）→ 加速（主升浪，戴维斯双击核心区）→ 减速（增速见顶回落）→ 反奇点（景气拐点确认）。两套框架数学本质相同，都指向"当前阶段是加速/左山坡"为最佳投资窗口。

### 位置感框架（广发策略）

从单纯的景气度追踪升级为"位置→空间→追踪→坐标"的体系化框架：

- **位置**：当前在周期的哪个阶段？（用山峰理论和成长股投资时钟分类）
- **空间**：如果位置有利，还有多少上行空间？（渗透率、估值分位、产能供给）
- **追踪**：什么信号会确认或证伪当前定位？（ΔG 更新、六维仪表盘、交期趋势）
- **坐标**：横向对比的参照系是什么？（同行业 peer、跨行业排名、历史周期对比）

## 与 davis_analyzer 的关系

`davis_analyzer` 包包含景气度评分引擎，本 skill 复用而非重写这些引擎：

| 引擎模块 | 核心功能 | skill 映射 |
|----------|---------|-----------|
| `prosperity.py` | 四维评分（营收+利润+斜率+持续时间）+ ΔG 计算 + 杜邦分解 | G+ΔG 框架的计算核心 |
| `prosperity_inflection.py` | 拐点识别 + 四阶段分类（加速/减速/上升拐点/下降拐点）| 成长股投资时钟的分类实现 |
| `prosperity_sector.py` | 行业聚合 + 阶段分类 + 二次点火筛选 + 相对 ΔG | 位置感框架的坐标计算和行业对比 |

使用规则：
- 调用引擎函数，不要复制引擎代码
- 不要修改引擎的权重、阈值或评分逻辑
- 六维指标数据需要手动收集，引擎不提供自动采集

## 核心原则摘要

- **ΔG 比 G 更重要**：增速的边际变化比增速绝对值对股价指引更强
- **斜率比位置更重要**：景气预期的变化速率比绝对水平更值得关注
- **30% 是关键阈值**：净利润增速降至 30% 以下时超额收益大概率下滑
- **六维交叉验证**：单一指标不可靠，六个维度同向上行才是最强信号
- **如实分类**：不能因为"行业景气"就强行说每只个股都在加速，行业 β 不等于个股 α
- **复用引擎**：调用 davis_analyzer，不重写评分逻辑

## 给 agent 的默认行为预期

当 agent 使用这套 skill 时，应该做到：

- 在给出周期定位结论前，先调用 davis_analyzer 计算景气度评分和 ΔG
- 周期位置分类必须如实，不能因为行业景气就强行说个股在加速期
- 六维指标数据需要手动收集并标注来源，引擎不提供自动采集
- 如果数据不足（少于 2 个季度），必须显式说明 ΔG 不可靠，而不是强行计算
- 不修改引擎源代码，通过调用函数完成评分和分类
- 30% 阈值、50% 降幅、2.5 年统计锚点都是参考基准，不是硬性规则，强产业趋势可以突破

换句话说，这套 skill 不是为了让景气度分析更乐观或更悲观，而是让分析过程更可追溯、更可比较、更可被质疑。

## 一句话总结

这套 skill 的本质，不是发明一种新的景气度分析公式，而是建立一种稳定的景气度投资治理方式：**G+ΔG 框架先行、六维指标交叉验证、山峰理论定性定位、引擎复用不重写、如实分类不强行说景气。**
