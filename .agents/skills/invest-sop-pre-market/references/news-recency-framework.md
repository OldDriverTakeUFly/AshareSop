# 消息时效性方法论：T0/T1/T2 三级框架

本文件定义盘前/盘后报告中对**消息面**的时效性处理规则。核心动机：一条消息越久远，就越不该作为**当日**预判的最终依据——应改看其后续实际走势（影响是否被消化/加剧）来辅助判断。任何暴跌/暴涨类消息在进入当日预判前，**必须**与近 3 日实际指数走势交叉核验。

> 适用范围：海外市场（美股/欧股/港股隔夜）、国内重大事件、政策、宏观数据发布、个股公告与新闻——所有以「消息」形式进入报告的内容。

## 1. 为什么需要这个框架

盘前报告的直接目的是**指导当日开盘前后的判断**。一个典型的失效模式：

> 某板块指数在 D-3 暴跌 7.87%。D 日盘前报告把这条消息当成当日「半导体大概率低开」的**核心依据**，却未核验 D-2、D-1 这两天该指数是否已经反弹修复。

后果：D 日实际开盘可能并未低开（冲击早已被消化），报告的预判落空。根因不是消息真假，而是**把过期消息当成了当日依据**，且没有任何机制要求核验「消化状态」。

本框架用两个机制堵住这个漏洞：
1. **时效性分级（T0/T1/T2）** —— 按消息年龄决定它在报告里的角色
2. **消化状态强制核验** —— 任何冲击类消息必须对照近 3 日实际走势判定「消化/加剧/中性」

## 2. T0/T1/T2 时效性分级

以**事件发生日**（event_date，消息所指事件真正发生的日期，不是 WebSearch 返回日）相对**报告日**（today）的日历天数划分：

| 层级 | 时间窗 | 在报告中的角色 | 可否作当日预判依据 |
|------|--------|----------------|:------------------:|
| **T0 当日核心** | today、today−1（最近一个完整交易日的盘中/盘后 → 开盘前） | 可作当日盘前**最终预判依据**，写入「今日重点」 | ✅ 可以 |
| **T1 趋势辅助** | today−2 ~ today−3 | **不可**作当日依据；引用时**必须**标注消化状态，放「趋势辅助」节 | ❌ 不可以 |
| **T2 背景参考** | today−4 及更早 | 仅作背景，**不进**当日预判逻辑，**不写入**「今日重点」 | ❌ 不可以 |

> **T0 的「today−1」含义**：指最近一个**已收盘**的交易日。例如 A 股 D 日盘前，T0 = D−1 夜盘美股（刚收盘的隔夜行情）+ D 日凌晨发生的政策/事件。「盘中/盘后」特指美股盘中及盘后（盘后财报如美光盘后 +16% 属 T0）。

### 边界速查（报告日为 D）

| 事件发生日 | 层级 | 一句话 |
|------------|------|--------|
| D、D−1 | T0 | 可作依据 |
| D−2、D−3 | T1 | 趋势辅助，须标消化状态 |
| D−4、D−7、更早 | T2 | 仅背景 |

**代码判定**：`stockhot.pre_market.classify_news_recency(event_date, today)` 返回 `tier` 与 `usage` 指引，边界由 `_T0_MAX_DAYS=1`、`_T1_MAX_DAYS=3` 控制。

## 3. 消化状态强制核验（核心规则）

**规则**：任何 T1（或被质疑的 T0）冲击类消息（暴跌/暴涨 ≥2%，或重大政策/事件）在写入报告前，**必须**与近 3 日实际指数走势交叉对照，判定三者之一：

| 状态 | 判定 | 处理 |
|------|------|------|
| **消化** | 后续走势已反向修复（窗口累计反向 ≥+1%） | 影响已计入，**不再据此预判当日**；如需提及，仅作「冲击已被消化」陈述 |
| **加剧** | 后续继续同向（窗口累计同向 ≤−1%） | 影响发酵中，**谨慎提示**，但**不作主导依据** |
| **中性** | 后续横盘（窗口累计 \|值\| <1%） | 影响不明，**标注「待观察」** |

**数据源**：`stockhot.pre_market.read_recent_overseas_trend(days_back=3)` 读取 `invest_overseas_market` 近 3 日行，返回 `OverseasTrendSummary`（含每日涨跌、窗口累计、VIX 走势、自动消化提示 `digestion_hint`）。

```python
from stockhot.pre_market import classify_news_recency, read_recent_overseas_trend
from datetime import date

today = date(2026, 6, 26)

# 1. 先分级
verdict = classify_news_recency("2026-06-23", today=today)   # days_old=3 -> T1
print(verdict.tier, verdict.can_be_today_basis)              # T1 False

# 2. T1 强制核验消化状态
trend = read_recent_overseas_trend(days_back=3, end_date=today)
print(trend.digestion_hint)   # 自动给出 消化/加剧/中性/待观察 提示
print(trend.sp500_cum, trend.nasdaq_cum, trend.vix_trend)    # 累计走势佐证
```

**当 `available_days == 0`**（窗口内无海外数据采集）：`digestion_hint` 会提示「消化状态无法判定」。此时该 T1 消息**不得**进入当日预判，只能标注「数据缺失，影响待核验」。

## 4. 报告各节放置规则

| 消息层级 | 放置位置 | 必须附带 |
|----------|----------|----------|
| T0 | 「今日重点」/ 综合判断预判依据 | 真实事件日期 + 来源 |
| T1 | 「趋势辅助」节（单独成节或并入综合判断的辅助段） | 真实事件日期 + **消化状态**（消化/加剧/中性）+ 近 3 日走势数据 |
| T2 | 省略；如确需提及，仅作背景一句话 | 标注「背景信息，距今 N 天，不作当日依据」 |

**反模式（禁止）**：
- 把 T1/T2 消息写进「今日重点」作为预判依据
- 引用 T1 冲击消息却不标消化状态
- 引用任何 ≥4 天前的暴跌/暴涨消息作为当日方向判断，且不附近期走势

## 5. 抽象化反例（本框架要防止的失效模式）

> 假设报告日为 D。D−3 某半导体指数暴跌 7.87%。报告在 D 日盘前写成：「该板块今日大概率低开」，依据正是这条 D−3 的暴跌消息——**却未核验 D−2、D−1 该指数的实际走势**。

失效点拆解：
1. **分级错位**：D−3 属 T1，本就不该作当日预判依据
2. **跳过消化核验**：未读取近 3 日走势；若 D−2、D−1 该指数已反弹修复（消化），则「低开」预判大概率落空
3. **未降级处理**：即便影响仍在，T1 也只应「谨慎提示」，不作主导

**正确写法**（假设近 3 日该指数累计已 +3.2%，即消化）：
> 趋势辅助（T1，D−3）：D−3 该指数暴跌 7.87%，但近 3 日累计 +3.2% 已反向修复，**冲击大概率已被消化**，不据此预判 D 日开盘方向。

## 6. 国内事件与政策消息

同样的 T0/T1/T2 规则适用于国内消息面，但消化核验的数据源不同：

| 消息类型 | 分级规则 | 消化核验数据源 |
|----------|----------|----------------|
| 国内重大事件（D−1 夜间发生） | T0 | 无需核验（当日新事件） |
| 国内政策（≥2 天前发布） | T1/T2 | 该政策相关板块近 3 日资金流（`invest_futures_sentiment` / 板块涨幅） |
| 宏观数据发布（如 PMI/CPI） | 发布日 T0，之后逐日降级 | 数据公布后市场反应（指数走势） |

**原则不变**：越久的国内消息，越要看其影响方向上的近期实际走势，而非直接拿消息本身判断当日。

## 7. WebSearch 来源的日期纪律

本框架依赖「事件发生日」准确。从 WebSearch 取消息时：

- **必须提取结果的实际发布日期**，填入引用的 `{日期}` 字段——不是报告日
- WebSearch 摘要常省略日期；优先点开来源页或看 URL 中的日期线索
- 拿不到确切日期的消息：按**最保守**估计（假设偏旧），归 T1 或 T2，不轻易当 T0
- 多个来源冲突时，以事件实际发生日（而非最早报道日）为准

## 8. 速查决策流（写盘前报告时对每条消息走一遍）

```
对每条隔夜/近期消息：
  1. 确定 event_date（事件实际发生日，非报道日）
  2. verdict = classify_news_recency(event_date, today)
       T0  -> 可写入「今日重点」作依据（附来源+日期）
       T1  -> 转步骤 3
       T2  -> 省略，或仅一句背景（标注距今天数，不作依据）
  3.（仅 T1）trend = read_recent_overseas_trend(days_back=3)
       读 digestion_hint + 累计走势，标「消化/加剧/中性」
       放「趋势辅助」节，不进「今日重点」
  4. 数据缺失（available_days==0）-> 标「消化状态无法判定」，不进预判
```

## Source of Truth

时效性边界（T0≤1 天、T1=2–3 天、T2≥4 天）与消化阈值（累计 ±1%、剧烈波动 ≥2%）为本框架的默认设定。若需调整，改 `stockhot/pre_market/__init__.py` 的 `_T0_MAX_DAYS` / `_T1_MAX_DAYS` 常量与 `_digestion_hint` / `read_recent_overseas_trend` 的阈值，本文件随之更新。判定逻辑以源码为准。
