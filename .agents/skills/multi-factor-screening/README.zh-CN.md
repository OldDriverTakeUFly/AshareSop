# 多因子量化选股 Skill 说明

这份文档是 `.agents/skills/multi-factor-screening/` 的中文说明版，面向人类读者。它帮助团队快速理解这套 skill 想解决什么问题、适合什么时候使用，以及如何把它部署到一次具体的选股任务里。

真正的规范性内容以 `SKILL.md` 为准。如果 agent 需要执行多因子选股相关工作，应优先读取 `SKILL.md`。本 README 仅是人类视角的概览，不构成规范性来源。

## 这套 skill 解决什么问题

从全市场数千只股票中筛选出值得关注的候选标的，本质上是一个信息过载问题。常见的痛点包括：

- 凭直觉选股，缺乏系统化的横向比较框架
- 因子选择缺乏经济学逻辑，过度依赖历史回测的统计显著性
- 把所有行业放在同一套权重下打分，导致"猎豹和老鹰赛跑"
- 忽略硬性风险过滤，选出的标的里混入 ST、高杠杆、现金流为负的高危股票
- 只看基本面不看资金情绪，错过 A 股市场最具特色的聪明钱信号
- 把选股当成择时，用因子模型做短线交易，信号时间尺度完全错配

这套 skill 的核心目标，是让 agent 在处理多因子选股时遵守同一套最小原则：

1. 先排除、后排序、再加分，三层逻辑不可跳过
2. 因子打分在行业内排序，不做全市场一刀切
3. 不同行业域使用不同权重，承认红利型和成长型的驱动逻辑不同
4. 加分层只做正向奖励，不出现稀缺信号不等于差公司
5. 只做当日截面排名，不做回测、不做 IC 分析、不做因子衰减曲线

## 何时使用

当遇到以下任务时，适合使用这套 skill：

- 从宽基股票池（沪深 300 + 中证 500，或全 A 非 ST）中筛选出候选清单
- 为特定域（红利型 / 成长型 / 价值型 / 周期型）构建关注列表
- 季度组合调仓时，重新评估和排序持仓候选
- 在行业内部做横向比较，选出该行业得分最高的标的
- 需要一套可复现、可审计的系统化选股流程

不适合使用的场景：

- 对单只股票做深度估值研究。多因子选股是横向比较工具，不能替代纵向深度分析。亏损标的深度估值请用 `valuation-loss-making-targets` skill
- 因子策略回测。这套 skill 只做当日截面排名，不做历史模拟
- 日内或高频交易。因子信号的时间尺度是月度到季度
- 极端行情下的择时。因子模型在股灾或泡沫期可能完全失效
- 纯主题或概念炒作驱动的行情。主题行情由叙事驱动，不是因子信号

## 如何部署

把这套 skill 部署到一次具体的选股任务，建议按以下顺序操作：

1. 先读 `SKILL.md`，理解三层结构（硬过滤 → 打分 → 加分）的设计逻辑和五大因子族的权重分配
2. 确定股票池。默认排除 ST、停牌、退市风险股。可以用宽基指数成分股作为起点
3. 按行业代码将股票池分为四个域：红利型、成长型、价值型、周期型
4. 执行硬过滤层。逐项检查流动性、ROE、CAGR、杠杆、现金流、估值，不合格的直接淘汰
5. 执行打分层。在行业内排序，用 0/1/2 三档法对每个因子打分，按域权重加权求和得到综合分
6. 执行加分层。检查机构加仓、高管增持、ESG 评级等稀缺信号，叠加加分（上限不超过综合分的 10%）
7. 输出四个域各自的排名清单，每域取前 N 名作为候选标的
8. 对候选标的做深度研究（阅读年报、行业分析、竞争格局），最终决定纳入组合的标的

部署时要注意：

- 因子打分必须在行业内做。银行 ROE 12% 在银行业是顶尖，在消费品行业可能只是中位数
- 域权重是硬编码的，不要为了"这次特殊"而随意调整。如果需要调整，应有明确的宏观分析依据
- 加分层信号缺位不扣分。没有机构加仓不等于公司不好，只是这个稀缺信号暂时没出现
- 排名清单是起点，不是终点。因子选股是"体检"和"面试"，不是"录用决定"

## 核心原则摘要

完整原则以 `SKILL.md` 为准，这里只做概述：

- **三层递进**：硬过滤层排除高风险标的，打分层区分优劣，加分层奖励稀缺信号。三层不可跳过、不可打乱
- **行业中性**：所有因子打分在行业内部排序，避免跨行业比较的失真
- **分域选股**：红利型看股息和现金流，成长型看增速和研发，价值型看估值和安全，周期型看 PB 和拐点。一域一权重
- **当日截面**：只做当下时点的排名快照，不碰历史回测和因子有效性监控
- **简洁优先**：5 个因子族、0/1/2 三档打分、4 个域。宁可牺牲精度也要避免过拟合

## 与 davis_analyzer 的关系

`davis_analyzer` 是本仓库自有的戴维斯双击估值分析引擎。这套 skill 和它的关系是：

| 维度 | multi-factor-screening | davis_analyzer |
|------|------------------------|----------------|
| 目的 | 横向排名筛选候选标的 | 纵向深度评分生成个股研报 |
| 输出 | 分域排名清单 | 每只标的的困境分、估值分位、景气度分 |
| 因子覆盖 | 5 族 4 域 | 4 维（估值、趋势、景气度、困境） |

这套 skill 会引用 `davis_analyzer` 的部分函数来获取数据和子分数（比如估值分位、景气度评分），但不修改其源码。两者权重体系不同：这套 skill 用 30/20/25/25，davis_analyzer 用 0.30/0.15/0.30/0.25。两套体系服务不同目的，不可混用。

## 给 agent 的默认行为预期

当 agent 使用这套 skill 时，应该做到：

- 先跑硬过滤层，绝不跳过直接打分
- 因子打分在行业内排序，用 0/1/2 三档法，不用连续分位数
- 按域应用不同权重，四个域分别输出排名清单
- 加分层只加不减，上限为综合分的 10%
- 不做回测，不算 IC，不画因子衰减曲线。只输出当日截面排名
- 如果数据缺失或来源不可靠，显式说明，不用替代值蒙混
- 排名清单附带说明：用了哪些因子、哪些域权重、哪些加分信号

换句话说，这套 skill 不是为了发明一种新的选股公式，而是建立一种稳定的选股治理方式：硬过滤兜底、行业中性打分、分域差异化加权、加分层捕捉稀缺信号、只做当日排名。

## 一句话总结

这套 skill 的本质，是把多因子选股从"黑箱"变成"三层管线"：硬过滤层保底线，打分层分优劣，加分层抓稀缺。所有权重硬编码，所有打分在行业内做，只输出当日截面排名清单。
