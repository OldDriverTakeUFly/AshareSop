# 研报质量检查清单

研报写作完成、提交推送前，逐项核对。这是 SKILL.md Phase 5（验证）的配套检查表。**所有项目通过后才算"验证完成"**，才能进入 commit/push。

## 结构完整性

- [ ] **标题含副标题**——副标题暗示结论方向（如"G+ΔG 框架下的下降拐点"）
- [ ] **开头元数据块齐全**——研究日期 + 股价/数据快照 + 核心逻辑（3-5 句）+ 具名数据来源
- [ ] **目录存在**且带锚点链接
- [ ] **章节齐全**——对照所选模板（A/B/C/D），无遗漏章节
- [ ] **无 `PART`/`待续`/`...未完`/`TODO` 标记**——这些出现在任何地方都意味着未完成
- [ ] **结尾固定结构齐全**——风险提示 + 免责声明 + 报告定位 + 版本历史表

## 数据可追溯性（最关键的 Tier-1 质量指标）

- [ ] **inline 来源标签密度 ≥80%**——随机抽 10 个数字，≥8 个带 `(value, source date)` 标签
- [ ] **数据来源索引表**（个股/产业链必备）——D1-Dn 编码表，每条带可靠度评级（高/中高/中）
- [ ] **交叉验证原则已声明**（必备）——在数据来源索引或方法论说明处写明"关键结论均采用 ≥2 个独立来源交叉验证"。这是 Tier-1 质量标志，缺此声明视为不完整
- [ ] **无来源不明的数字**——每个数字要么有具名来源，要么明确标注"估算"或"不可用"
- [ ] **未编造数据**——信息缺乏处标注"估算/不可用"，而非凭空填补

## 数据时效性（SKILL.md Phase 2 步骤 4.5）

- [ ] **取数前跑了时效性校验？**——确认 daily_basic 最新交易日 / income 最新报告期+披露日 / forecast 业绩预告（见 `references/engine-usage.md` §10）
- [ ] **财务快照是最新已披露季报？**——半年报 8 月底、年报 4 月底才披露，中途只能到上一季报；若非报告日数据，已先取最新
- [ ] **时效边界写进报告？**——标注「财务截至 YYYYQN（YYYYMMDD 披露），下季报 X 月披露」；估值快照标注交易日
- [ ] **业绩预告已纳入？**（若有）——H1/全年预告常先于正式财报，是领先信号，不能漏

## 财务与景气度分析

- [ ] **多年财务轨迹表**——个股报告：≥4 季度单股财务表（营收/净利/增速/毛利率/ROE）；产业链报告：≥1 张多标的财务速查对比表
- [ ] **拐点诊断有成因**——不只是"业绩下滑"，而是列出具体原因（价格/销量/费用/一次性）
- [ ] **G 和 ΔG 同时给出**（个股报告必备，产业链报告对焦点标的做）——景气度章必须同时有增速绝对值和边际变化
- [ ] **山峰理论定位**（个股报告必备）——Δg 符号 → 左山坡/右山坡/山后的明确判定
- [ ] **30% 阈值检验**——净利润增速是否跌破 30%，超额收益风险是否标注
- [ ] **股东户数趋势已分析？**（个股报告必备）——`stk_holdernumber` 取近 4-8 期，算环比变化并判断集中/分散趋势（户数降=筹码集中=动能增强✓；户数升=分散=动能减弱⚠）。模板见 `references/engine-usage.md` §11

## 估值分析

**估值要求按报告类型不同——个股报告做完整估值，产业链报告做横向估值对比。**

### 个股研报（模板 A）——完整估值（全部必备）

- [ ] **历史估值分位表**——近 3 年 PE/PB/PS，10/25/50/75/90/95 分位 + 当前值
- [ ] **≥2 个估值模型**——PE/PB/PS 三角验证（盈利标的）或 PS/DCF 双模型（亏损标的）
- [ ] **三情景区间**——悲观/中性/乐观，每个有隐含股价 + vs 当前涨幅
- [ ] **概率加权目标价**——展示加权公式（如 20%×悲观 + 50%×中性 + 30%×乐观）
- [ ] **禁止单一目标价**——这是硬规则，违反即不合格
- [ ] **亏损标的禁用 PE 主估值**——用 PS/DCF，PE 仅作诊断辅助
- [ ] **三模型收敛/发散判断**——若发散，明确标注为风险信号

### 产业链研报（模板 B）——横向估值对比（无需个股级三模型）

- [ ] **标的估值横向对比表**——产业链内全部 A 股标的的 PE/PB/分位/市值/ROE 一表呈现，标出估值最合理/最极端的标的
- [ ] **估值筛选结论**——基于分位 + 基本面给出"哪些标的估值合理、哪些高估"的判断（不是单一目标价）
- [ ] **焦点标的可加深**——若产业链报告聚焦某只标的（如 AIDC 报告的麦格米特），该标的部分可按个股要求做景气度分析，但**不强制完整三模型**

### 方法论研报 / 分析笔记（模板 C/D）

估值章节非强制——方法论报告无具体标的；分析笔记按需简述。

### 所有类型通用

- [ ] **禁止单一目标价**——任何涉及目标价的场合，必须是区间或情景加权

## 竞争与对比

- [ ] **至少一个横向对比表**——≥3 家可比（国内+国际+未上市/IPO 的），多维度列对比
- [ ] **可比公司真实可查**——不编造，财务数据可访问
- [ ] **投资逻辑链**——因果链（景气根基→景气强度→周期定位→α来源→盈利弹性→标的选择），不是要点罗列

## 合规与诚实

- [ ] **无买卖建议**——表述为"矩阵结果"/"投资框架定位"，不写"建议买入/卖出"
- [ ] **风险提示具体**——≥4 条编号风险，不是泛泛的"市场风险"
- [ ] **研究局限诚实声明**——标注估算/不可用项，"未做任何编造"
- [ ] **免责声明齐全**——AI 辅助 + 不构成投资建议 + 风险枚举 + "市场有风险，投资需谨慎"

## 差异化（若 docs/ 已有同标的报告）

- [ ] **边界声明**——明确声明本报告与既有报告的差异/新增内容
- [ ] **非简单复制**——表格数据重新取数，不复制旧报告
- [ ] **版本整合**（若同标的多份）——优先整合为主报告 + 附录存档

## 提交前最后检查

- [ ] **中文文件名**——非英文（如 `docs/钽电容产业链深度研报.md`）
- [ ] **放入正确目录**——个股研报/{行业}/、产业链研报/、方法论/、分析笔记/
- [ ] **docs/README.md 索引已更新**——新报告登记到对应分类表
- [ ] **引用了 docs/方法论/ 的相关文档**（如适用）——形成知识图谱

## 判定

- [ ] **全部通过** → 进入 commit + push（SKILL.md Phase 6）
- [ ] **数据缺失 >25%** → 停止，向用户报告数据缺口，不粉饰为完整
- [ ] **估值输出单一目标价** → 必须改为概率加权区间后才能提交
- [ ] **来源密度 <80%** → 补充来源标注，或标注"估算"，达到阈值再提交
