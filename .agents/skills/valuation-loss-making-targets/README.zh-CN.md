# 亏损标的估值 Skill 说明

这份文档是 `.agents/skills/valuation-loss-making-targets/` 的中文说明版，面向人类读者。它的目标不是替代 `SKILL.md`，而是帮助团队快速理解这套 skill 想解决什么问题、适合什么时候使用，以及如何把它部署到一次具体的并购 / 研究任务里。

真正的规范性内容仍以 `SKILL.md` 为准。如果 agent 需要执行亏损标的估值相关工作，应优先读取 `SKILL.md` 和配套 checklist、references 文件。本 README 仅是人类视角的概览，不构成规范性来源。

## 这套 skill 解决什么问题

对亏损企业做估值，本质上比盈利企业难得多。常见痛点包括：

- 亏损公司无法直接套用 PE、PEG 这类依赖正向净利润的倍数
- 用 PS、EV/Revenue 这类收入倍数时，很容易忽视亏损背后的结构性问题
- 同行业可比公司筛选粗放，导致标的被放到一个"看起来差不多、实际上完全不同"的样本组里
- 困境企业有真实的破产 / 退市 / 资金链断裂风险，但估值模型经常把这种概率当作 0
- 研究脚本和数据来源不透明，结论无法被复现，也无法被质疑
- 财务深度只看了三五个指标，就把一个复杂的亏损生意讲完了

这套 skill 的核心目标，是让 agent 在处理亏损标的估值时遵守同一套最小原则：

1. 先做差异化审计，再谈可比性
2. 数据来源必须可追溯，每一个数字都能回到原始 source
3. 困境概率必须显式建模，不能默认为零
4. 估值模型要按统一 schema 输出，便于横向比较
5. 财务深度要覆盖足够多的维度，而不是挑几个好看的指标

## 何时使用

当遇到以下任务时，适合使用这套 skill：

- 评估一家仍在亏损的公司作为潜在收购标的
- 对一个陷入经营困境、但具备战略价值的标的公司做估值
- 需要在多个亏损标的之间做横向对比并给出排序
- 给投研团队建立一套可复用的亏损标的估值流程
- 对外部研究报告中的亏损公司估值做交叉验证
- 需要把量化数据获取、评分、财务深度分析串成一条可复现的 pipeline

不适合使用的场景：

- 标的公司已经稳定盈利，且不涉及困境假设 —— 直接用常规估值方法即可
- 纯定性的战略讨论，不需要量化结论
- 数据源完全缺失，连基础财务报表都拿不到

## 如何部署

把这套 skill 部署到一次具体的估值任务，建议按以下顺序操作：

1. 先读 `SKILL.md`，理解核心方法论、agent contract 和工作流
2. 用 `checklists/differentiation-audit.md` 对标的做 5 轴差异化审计，确认它和候选可比公司到底差在哪里
3. 用 `checklists/source-traceability.md` 逐项核对数据来源，确保每一个输入数字都可追溯
4. 按 `references/valuation-model-schema.md` 的规范搭建估值模型，保证输出结构统一
5. 按 `references/distress-probability-rules.md` 把困境概率显式纳入估值调整
6. 用 `references/study-script-templates/` 下的模板脚本完成量化数据获取、评分和财务深度分析
7. 把整个流程的输入、来源、假设、结论记录下来，形成可复现的研究产物

部署时要注意：

- 模板脚本是参数化的，需要根据具体标的填充 ticker、时间窗口、行业代码等参数
- schema 是强约束，不要为了"这次特殊"而临时改字段结构
- 困境概率不是拍脑袋的数字，必须依据规则文档推导

## 核心原则摘要

完整原则以 `SKILL.md` 为准，这里只做概述：

- **差异化优先于可比性**：亏损标的之间的差异往往比盈利公司更大，不审计就谈可比性是危险的
- **来源优先于结论**：一个无法追溯到 source 的数字，在估值里没有意义
- **困境概率优先于乐观假设**：亏损公司最常见的问题不是"能不能赚钱"，而是"能不能活下来"
- **结构统一优先于灵活性**：所有标的共用一套 schema，才能横向比较
- **深度优先于数量**：宁可覆盖八个关键指标，也不要只看三个但讲得很精彩

## 相关文件

- `SKILL.md`
  - 规范性主文件（核心方法论，事实来源）。定义 agent contract、估值原则、工作流和判断规则。本 README 的所有内容均服从于此文件。
- `checklists/differentiation-audit.md`
  - 5 轴差异化审计清单。用于在选可比公司之前，先把标的和候选样本的结构性差异讲清楚。
- `checklists/source-traceability.md`
  - source 可追溯验证清单。用于核对估值模型的每一个输入数字是否都能回到原始来源。
- `references/valuation-model-schema.md`
  - 估值模型 schema 规范。定义统一的输出结构，保证多个标的之间可以横向比较。
- `references/distress-probability-rules.md`
  - 困境到概率的调整规则。定义如何把经营困境显式转化为破产 / 退市概率，并纳入估值。
- `references/study-script-templates/quant_data_template.py`
  - 参数化数据获取脚本模板。用于按统一方式抓取标的的量化数据。
- `references/study-script-templates/scoring_template.py`
  - 参数化评分适配器模板。用于把多维指标映射到统一评分。
- `references/study-script-templates/financial_deep_template.py`
  - 参数化 8 指标财务深度脚本模板。用于对标的公司做结构化的财务深度分析。

## 给 agent 的默认行为预期

当 agent 使用这套 skill 时，应该做到：

- 在给出估值结论前，先完成差异化审计和来源追溯
- 困境概率必须有据可查，不能默认为零或随意填一个数
- 所有模型输出严格遵循 schema，不私自加字段或改字段含义
- 模板脚本只参数化、不改逻辑；如需扩展，应在 SKILL.md 里登记
- 如果数据缺失或来源不可靠，必须显式说明，而不是用替代值蒙混过去

换句话说，这套 skill 不是为了让估值更乐观或更悲观，而是为了让亏损标的的估值过程更可追溯、更可比较、更可被质疑。

## 一句话总结

这套 skill 的本质，不是发明一种新的估值公式，而是建立一种稳定的亏损标的估值治理方式：**差异化审计先行、来源可追溯、困境概率显式建模、模型结构统一、财务深度足够。**
