# CodeAgent Dashboard

CodeAgent Dashboard 是一个面向人机协作的软件项目看板系统，目标是把任务规划、进度展示、任务创建、审批控制，以及 AI Agent 执行能力放到同一个工作台中。它不是单纯的待办列表，也不是只会触发脚本的自动化面板，而是一个以任务为核心、以执行过程可追踪为前提的管理平台。

项目的核心问题是，团队在使用 AI Agent 执行研发或运营任务时，往往缺少统一入口来回答这些问题：

- 现在有哪些任务，谁创建的，优先级是什么
- 哪些任务正在执行，执行到哪一步，是否阻塞
- 哪些操作需要人工审批，哪些结果已经被接受或拒绝
- 每次 Agent 执行做了什么，为什么这样做，是否留下了完整审计记录
- 哪类任务适合串行执行，哪些可以并行拆分以加快完成速度

本项目希望提供一个实用、可演进的答案。MVP 先聚焦最小闭环，用户可以创建任务、查看状态、触发执行、在必要节点进行审批，并查看清晰的审计轨迹。系统采用模块化单体架构，在一个部署单元内完成主要业务拆分，控制复杂度，同时为后续拆分服务保留边界。

## 项目目标

这个仓库的目标是支撑一个 Dashboard 产品，覆盖以下能力：

- 规划任务，维护任务标题、描述、优先级、负责人、目标时间等信息
- 展示任务和执行进度，包括待开始、执行中、待审批、已完成、失败、取消等状态
- 创建并分派任务给 AI Agent 执行，支持按 Agent 能力选择执行者
- 管理审批流程，让高风险步骤或最终结果先经过人工确认
- 保存完整审计轨迹，包括任务变更、调度决策、Agent 执行动作、审批记录
- 汇总执行分析数据，为后续优化任务拆分、Agent 选择和流程设计提供依据

## 核心设计原则

### 1. 任务定义与执行记录分离

系统明确区分 `Task` 和 `TaskRun`：

- `Task` 表示一项待完成的业务目标，是用户关心的对象
- `TaskRun` 表示一次具体执行尝试，是系统调度和审计的对象

这种分离可以让同一个任务多次执行、重试、改派 Agent、保留历史，不会把业务目标和运行过程混在一起。

### 2. 先支持清晰闭环，再扩展智能协作

MVP 不追求复杂自治，而是先把任务创建、调度、执行、审批、追踪这一条链路做稳。系统应优先做到可解释、可恢复、可审计。

### 3. 并行能力从第一版就要实用

对于彼此独立的任务或子任务，MVP 推荐支持并行分派给多个 Agent，以缩短总完成时间。比如一个需求可以拆成文档撰写、数据调研、测试用例生成三个独立子任务，并交给不同 Agent 同时执行。

这里需要明确一个实现边界：在 MVP 中，子任务一律建模为有 `parentTaskId` 的子 `Task`，每个子 `Task` 再拥有自己的 `TaskRun`。系统支持“多个 Agent 同时处理多个独立工作项”，但不支持“多个 Agent 同时协作同一个运行实例”。

但更复杂的多 Agent 协作，例如共享上下文协商、动态辩论、联合决策、跨 Agent 自动移交，不必在 MVP 完成。v1 可以在稳定的调度和审计基础上，再增加更高级的协作机制。

### 4. 审批与审计是产品的一部分

审批不是附属功能，审计也不是仅供排障的日志。两者都属于产品主线，因为 AI Agent 执行带来的价值，必须以可控和可追溯为前提。

为控制 MVP 复杂度，第一版默认只提供“最终结果验收审批”这一条主审批链路：如果任务被标记为需要审批，则运行成功后进入待审批；审批通过后任务完成，审批拒绝后任务回到可重新处理状态。更细的开始前审批和执行中高风险动作审批留给后续版本按需扩展。

## MVP 预期能力

MVP 面向一个真实可用的基础工作台，最少应包含：

- Dashboard 首页，展示任务总览、运行中任务、待审批事项、近期执行结果
- 任务管理页面，支持创建任务、编辑任务、查看详情、手动触发执行
- 执行记录页面，展示每个 `TaskRun` 的状态、开始时间、结束时间、关联 Agent、输出摘要、失败原因
- 审批页面，支持查看待审批项并执行批准或拒绝
- 审计记录页面，查看关键事件时间线
- 基础分析能力，至少能看到任务完成率、平均执行时长、失败率、审批等待时长等指标

## 适用场景

这个项目适合以下场景：

- 团队希望系统化地管理 AI Agent 执行的软件任务
- 任务经常需要人工确认，不能完全放任自动执行
- 一个目标常常能拆分成多个可并行处理的子任务
- 团队希望在保留执行效率的同时，持续积累可分析的数据和审计记录

## 版本演进建议

### MVP

- 单仓库、单部署单元的模块化单体
- 明确的 `Task`、`TaskRun`、`Approval`、`AuditEvent` 模型
- Agent Registry 支持注册和按能力筛选 Agent
- Orchestrator 支持串行与独立子任务并行调度
- 基础 Analytics 报表

### v1

- 更细的任务拆分策略
- 更丰富的多 Agent 协作模式
- 基于历史表现的 Agent 选择优化
- 更强的审批策略配置
- 面向团队的权限与操作边界控制
