# A股每日热点分析工具 - 项目需求规格说明书

## 1. 项目概述

**项目名称**: A股热点追踪 (StockHot-CN)

**核心定位**: 针对A股市场的每日热点分析工具，自动采集市场数据、AI智能分析热点主题、生成可视化报告图片，用于社交媒体发布与后台数据存储。

**目标用户**: 个人投资者（需要易于理解的热点解读）

---

## 2. 功能需求

### 2.1 数据采集模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 每日涨跌幅排行 | 获取当日涨幅前N/跌幅前N的股票 | P0 |
| 板块/概念轮动 | 获取当日热门板块、概念涨幅排行 | P0 |
| 资金流向 | 获取主力资金净流入/流出排行 | P1 |
| 涨停板分析 | 获取当日涨停股、连板情况 | P1 |
| 龙虎榜数据 | 获取机构/游资买卖前列 | P2 |
| 公告/新闻采集 | 获取影响市场的重大公告、新闻 | P1 |

**数据源策略**: 
- 初期：使用免费接口（如Tushare免费接口、东方财富Choice免费接口）
- 后续：可扩展付费数据源

### 2.2 AI智能分析模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 热点归因 | AI分析当日热点背后的原因（政策、行业周期、资金推动等） | P0 |
| 主题聚类 | 将零散股票归类到统一主题/概念 | P0 |
| 趋势预判 | 基于历史数据给出短期趋势参考 | P1 |
| 每日报告生成 | 自动生成结构化每日市场分析报告 | P0 |
| 风险提示 | 识别潜在风险点（如过度炒作、估值泡沫） | P1 |

### 2.3 图片生成模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 小红书尺寸适配 | 生成适配小红书的图片尺寸 (1242x1660 推荐) | P0 |
| 数据卡片生成 | 将关键数据生成为美观的图片卡片 | P0 |
| 热点榜单图 | 涨幅榜/跌幅榜/资金流榜等图片 | P0 |
| AI分析报告图 | 将AI生成的分析报告转为图片 | P0 |
| 自定义模板 | 支持不同风格的图片模板 | P1 |
| 封面图生成 | 生成吸引眼球的封面图用于笔记 | P0 |

### 2.4 社交媒体发布模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 定时任务 | 支持每日收盘后自动运行 | P0 |
| 小红书发布 | 支持自动发布到小红书 | P0 |
| 发布记录 | 记录历史发布内容与时间 | P1 |

### 2.5 数据存储与分析模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 历史数据存储 | 存储每日市场数据供回溯分析 | P0 |
| 自选股跟踪 | 关注特定股票/板块的分析 | P1 |
| 个性化推送 | 根据用户关注领域定制内容 | P2 |
| 数据导出 | 支持导出Excel/CSV | P1 |

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 稳定性 | 每日任务成功率 > 99% |
| 时效性 | 收盘后2小时内完成全流程 |
| 可扩展性 | 插件化数据源、图片模板 |
| 数据安全 | 本地存储、不上传敏感信息 |
| 成本控制 | 初期使用免费接口，控制成本 |
| 数据保留 | 历史数据保留90天，自动清理过期数据 |

## 5. 技术架构建议

### 5.1 推荐技术栈

| 层级 | 推荐方案 |
|------|----------|
| 后端语言 | Python（数据处理、AI集成优势） |
| 数据存储 | SQLite（轻量）+ PostgreSQL（可选） |
| AI能力 | OpenAI API / Claude API / 本地LLM |
| 图片生成 | Pillow + SVG模板 / Canva API |
| 定时任务 | APScheduler / Cron |
| 部署 | Docker + 云服务器 |

### 5.2 模块划分

```
stockhot-cn/
├── data_collector/      # 数据采集模块
│   ├── tushare_client/   # Tushare接口
│   ├── eastmoney_client/# 东方财富接口
│   └── news_client/     # 新闻/公告采集
├── ai_analyzer/         # AI分析模块
│   ├──热点归因/
│   ├──主题聚类/
│   └──报告生成/
├── image_generator/     # 图片生成模块
│   ├── templates/      # 小红书风格模板
│   │   ├── cover/      # 封面图模板
│   │   ├── data_card/  # 数据卡片模板
│   │   └── report/    # 报告模板
│   └── renderer/       # 渲染引擎
├── publisher/           # 发布模块
│   └── xiaohongshu/   # 小红书发布
├── storage/            # 数据存储
│   ├── database/       # SQLite/PG
│   └── files/          # 图片/报告存储
└── scheduler/          # 定时任务调度
```

## 6. MVP 范围（首版发布）

### 必须完成 (P0)

1. **数据采集**: 每日涨跌幅排行 + 板块排行 + 资金流向
2. **AI分析**: 热点归因 + 每日报告生成
3. **图片生成**: 核心数据卡片 + 热点榜单图
4. **定时运行**: 每日收盘后自动执行
5. **本地存储**: 历史数据保存

### 后续迭代 (P1-P2)

- 社交媒体自动发布
- 个性化订阅
- 更多数据源接入

## 7. 风险与挑战

| 风险 | 应对方案 |
|------|----------|
| 免费API不稳定 | 设计降级方案、备选数据源 |

| 图片版权问题 | 使用自主设计模板 |
| 合规风险 | 不提供投资建议、仅展示数据 |

## 10. 小红书平台适配说明

### 10.1 图片规格

| 类型 | 推荐尺寸 | 比例 | 用途 |
|------|----------|------|------|
| 封面图 | 1242x1660 | 3:4 | 笔记封面，最佳显示 |
| 内容长图 | 1240x自适应 | - | 完整报告、数据卡片 |

### 10.2 内容风格（数据风）

- **配色方案**: 深色背景(#1a1a2e) + 亮色数据(#00d4ff) + 警示红(#ff6b6b) + 增长绿(#00c853)
- **字体**: 思源黑体/Noto Sans SC，数据加粗突出
- **图表**: 柱状图、折线图、饼图结合，颜色鲜明
- **布局**: 卡片式设计，信息层次分明

### 10.3 发布策略

- 发布时间：工作日 16:00-18:00（收盘后）
- 封面标题：吸引眼球 + 关键词
- 话题标签：#A股 #每日复盘 #热点追踪 #投资
