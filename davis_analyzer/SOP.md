# 戴维斯双击估值分析器 — 标准作业流程（SOP）

本文档记录 `davis_analyzer` 的全部数据获取规则与计算方法。所有权重、阈值均来源于 `constants.py`，算法逻辑来源于各引擎模块。

---

## 第一章：数据获取规则

### 1.1 数据源

仅使用 Tushare Pro API，涉及以下接口：

| 接口 | 用途 | 关键字段 |
|------|------|----------|
| `stock_basic` | 股票列表 | ts_code, name, industry, list_status |
| `daily_basic` | 每日估值指标 | ts_code, trade_date, pe_ttm, pb, ps, total_mv |
| `income` | 利润表 | ts_code, end_date, total_revenue, n_income |
| `balancesheet` | 资产负债表 | ts_code, end_date, total_assets, total_liab |
| `cashflow` | 现金流量表 | ts_code, end_date, n_cashflow_act |
| `fina_indicator` | 财务指标 | ts_code, end_date, roe, eps, dt_eps, revenue_ps |

### 1.2 频率限制

`TUSHARE_RATE_LIMIT = 500`（每分钟最多 500 次调用）。客户端采用滑动窗口限速：60 秒窗口内请求数达到上限时自动等待。

失败重试：最多 3 次（`_MAX_RETRIES = 3`），指数退避，基准 1 秒。

### 1.3 缓存机制

SQLite 缓存，表名 `api_cache`，主键为 `(endpoint, params_hash)`。

- params_hash：对请求参数做 JSON 序列化后取 SHA-256 摘要
- 无 TTL（无过期时间），缓存永久有效
- 空 DataFrame 不写入缓存

### 1.4 估值回溯周期

`PERCENTILE_DAYS = 1095`（3 年自然日）。从当前日期回溯 1095 天，获取 `daily_basic` 日线数据用于估值分位计算。

### 1.5 预筛规则

构建股票宇宙时排除以下标的：

- 名称包含 `ST` 或 `*ST`（`EXCLUSION_PATTERNS`）
- 上市状态非 `L`（排除停牌、退市股）

### 1.6 周期性行业

`CYCLICAL_INDUSTRIES` 包含 7 个行业，这些行业的股票标记为 `is_cyclical = True`，估值引擎自动切换为 PB 替代 PE：

```
钢铁, 有色金属, 煤炭, 石油石化, 化工, 建材, 造纸
```

## 第二章：估值分位计算

### 2.1 历史百分位方法

采用"分数秩"（fractional rank）法：统计历史值中小于等于当前值的个数，除以总数。

- 空历史数据 → 返回 0.5（中性）
- 当前值 = 最小值 → 百分位 0.0
- 当前值 = 最大值 → 百分位 1.0

### 2.2 评分逻辑

PE/PB 数据缺失（NaN）的行被跳过。取最新有效行作为当前值。

**三种模式：**

1. **周期股**：`detect_cyclical(industry) == True` → 仅使用 PB 分位
2. **负 EPS 退化**：任一历史 PE < 0 或 PE < `EPS_NEAR_ZERO_THRESHOLD`（0.01） → 仅使用 PB 分位
3. **常规股**：PE 分位 × 0.6 + PB 分位 × 0.4（加权混合）

**评分公式：**

- PB 模式：`score = (1.0 - pb_percentile) × 100`
- 混合模式：`score = ((1.0 - pe_percentile) × 0.6 + (1.0 - pb_percentile) × 0.4) × 100`

分位越低（越低估），得分越高。评分范围 0-100。历史数据为空时返回 50.0（中性）。

## 第三章：PE/PB 趋势计算

### 3.1 数据准备

取最近 `PE_PB_TREND_MONTHS = 36` 个月（3 年）的日线数据，按月重采样取月均值，得到 36 个月度数据点。

### 3.2 趋势斜率

对月度序列做一阶线性回归（`numpy.polyfit` degree=1），取回归斜率。

### 3.3 加速度

对月度序列做二阶差分（二阶差分 = 当前差分 - 上一期差分），取全部二阶差分的均值。

### 3.4 评分映射

斜率映射规则：

- 斜率在 `[-1, 0]` 区间 → 分数 `[100, 50]`（线性映射，下行趋势越缓分数越高）
- 斜率 `> 0`（上行）→ 分数 `[50, 0]`（上行越快分数越低）

> 注：戴维斯双击策略寻找估值下杀的标的，因此下行趋势（PE/PB 持续走低）反而获得高分，代表估值正在收缩。

### 3.5 数据量门槛

`MIN_TREND_MONTHS = 12`。月度数据点少于 12 个时，趋势分数固定为 50.0（中性），不参与计算。

### 3.6 周期股处理

周期股仅使用 PB 月度序列计算趋势，不使用 PE。

## 第四章：景气度评分

### 4.1 四维加权结构

| 维度 | 权重 | 计算方法 |
|------|------|----------|
| 营收增长（revenue） | 0.30 | 近季度营收同比增速加权评分 |
| 盈利增长（profit） | 0.30 | 近季度净利润同比增速加权评分 |
| 趋势斜率（slope） | 0.25 | 营收增速序列线性回归斜率评分 |
| 持续时间（duration） | 0.15 | 连续正增长季度数评分 |

> 来源：`PROSPERITY_WEIGHTS = {revenue: 0.30, profit: 0.30, slope: 0.25, duration: 0.15}`

### 4.2 增长评分区间

单个增速值映射为 0-100 原始分（`_growth_to_raw_score`）：

| 增速区间 | 原始分 |
|----------|--------|
| > 30% | 80-100 |
| 10%-30% | 50-80 |
| 0%-10% | 25-50 |
| < 0% | 0-20 |

多季度数据按指数衰减加权（`decay = 0.8`），近期季度权重更高。

### 4.3 ΔG 边际变化

```
ΔG = 当前季度增速 - 上一季度增速
```

ΔG > 0 表示增速加速（看多），ΔG < 0 表示增速减速（看空）。

### 4.4 持续时间评分

连续正增长季度数：

| 连续季度数 | 得分 |
|------------|------|
| 0 | 0 |
| 1 | 25 |
| 2 | 50 |
| 3 | 75 |
| 4+ | 100 |

### 4.5 杜邦分解

```
ROE = 净利率 × 总资产周转率 × 杠杆倍数
```

取三者绝对值最大者为主要驱动因素，分类为：

- **净利率驱动（定价权强）**：净利率绝对值最大
- **周转率驱动（需求/效率提升）**：周转率绝对值最大
- **杠杆驱动（需警惕可持续性）**：杠杆倍数绝对值最大

### 4.6 信号质量

至少需要 2 个季度数据方可计算。数据不足 2 季度的股票在 `batch_prosperity` 中被跳过。

## 第五章：困境反转评分

### 5.1 三层信号体系

困境评分由三层信号构成，每层 3 个信号，共计 9 个信号。

**第一层：困境确认（权重 0.3）**

判断股票是否真正处于困境。

| 信号 | 判断逻辑 |
|------|----------|
| `eps_decline` | 最新季度 EPS 同比下滑 > 30%（同比比较：index [0] vs [4]） |
| `pe_pb_percentile` | PE 或 PB 百分位 < 10%（深度低估） |
| `financial_health` | 资产负债率 < 50% 且经营性现金流 > 0 |

**第二层：反转可能（权重 0.3）**

判断资产负债表是否支撑困境反转。

| 信号 | 判断逻辑 |
|------|----------|
| `balance_sheet` | 总负债 / 总资产 < 50% |
| `operating_cf` | 经营性现金流 > 0 |
| `roe_trend` | ROE 企稳或改善（同比比较：index [0] vs [4]，最新一期 ≥ 上年同期） |

**第三层：反转激活（权重 0.4）**

判断基本面是否出现拐点信号。

| 信号 | 判断逻辑 |
|------|----------|
| `revenue_inflection` | 营收增速由负转正（上期 < 0，当期 ≥ 0） |
| `profit_inflection` | 利润增速由负转正（上期 < 0，当期 ≥ 0） |
| `delta_g_positive` | ΔG > 0（增速加速） |

### 5.2 连续评分

每个信号产生 0.0-1.0 的连续得分，而非简单的二元（命中/未命中）。信号越接近触发条件，得分越高。

### 5.3 层级与总分计算

```
层级得分 = Σ(信号得分) / 信号数 × 100
总分 = 第一层 × 0.3 + 第二层 × 0.3 + 第三层 × 0.4
```

### 5.4 同比比较规则

`eps_decline` 和 `roe_trend` 采用同比比较（index [0] vs [4]），而非相邻季度比较，以消除季节性波动干扰。

## 第六章：戴维斯双击综合评分

### 6.1 四维加权

| 维度 | 权重 |
|------|------|
| 估值分位（valuation） | 0.30 |
| PE/PB 趋势（trend） | 0.15 |
| 景气度（prosperity） | 0.30 |
| 困境反转（distress） | 0.25 |

> 来源：`DAVIS_DOUBLE_WEIGHTS = {valuation: 0.30, trend: 0.15, prosperity: 0.30, distress: 0.25}`

四项权重之和为 1.0。

### 6.2 排序规则

按 `final_score` 降序排列，取前 N 名（默认 Top 30）。排名从 1 开始（1-based）。

### 6.3 投资结论分级

| 综合评分 | 结论 |
|----------|------|
| ≥ 80 | 强烈推荐关注 |
| ≥ 65 | 推荐关注 |
| ≥ 50 | 可关注 |
| < 50 | 谨慎观察 |

## 第七章：深度调研工作流

### 步骤 1：运行筛选

执行完整 8 步管线（见第八章），获取 Top N 标的列表。

### 步骤 2：生成调研清单

```bash
python -m davis_analyzer.cli deep-research --top N
```

为每只标的生成空白调研清单。

### 步骤 3：人工填写清单

人工补充以下维度的定性信息：

- 行业政策变化
- 机构观点与一致预期
- 近期重大公告
- 竞争格局分析

### 步骤 4：重新评分

工具读取已填写的清单，重新计算景气度评分和困境反转评分，融合定性信息。

### 步骤 5：生成最终报告

输出重新评分后的深度研报和汇总索引。

## 第八章：报告生成规则

### 8.1 管线 8 步结构

| 步骤 | 说明 | 产出 |
|------|------|------|
| Step 1/8 | 初始化 TushareClient | 客户端实例 |
| Step 2/8 | 构建股票宇宙（预筛 ST/停牌） | 约 4500 只 |
| Step 3/8 | 获取全市场估值数据（3 年日线） | 估值分位字典 |
| Step 4/8 | 粗筛：估值分数 > 50 | 约 500-800 只 |
| Step 5/8 | 获取粗筛股财务数据（4 张报表合并） | FinancialData 列表 |
| Step 6/8 | 计算景气度评分（四维加权） | ProsperityScore |
| Step 7/8 | 计算困境信号评分（三层加权） | DistressSignal |
| Step 8/8 | 计算戴维斯双击综合评分并排序 | DavisDoubleScore |

### 8.2 研报结构（7 章）

每只标的的深度研报包含：

1. **公司概况**：名称、代码、行业、市值、PE/PB
2. **估值分析**：PE/PB 百分位、估值判断
3. **景气度分析**：四维得分、ΔG、杜邦结论
4. **困境反转信号**：三层得分、信号明细
5. **戴维斯双击评分**：四维加权得分、排名
6. **风险因素**：数据局限性、行业风险
7. **投资结论**：按综合评分自动分级

### 8.3 文件命名

- 个股研报：`{排名}_{股票代码}_{股票名称}_深度研报.md`
- 汇总索引：`戴维斯双击估值筛选汇总_YYYYMMDD.md`

### 8.4 技术规格

- 每篇研报上限约 1500 词（`REPORT_MAX_WORDS = 1500`）
- 纯模板 + 数据插值生成，不依赖 LLM
- 计算来源说明格式：

```
> 计算方式：估值分位 = (1.0 - PB百分位) × 100（周期股PB模式）
> 计算方式：综合评分 = 估值×0.30 + 趋势×0.15 + 景气度×0.30 + 困境×0.25
```
