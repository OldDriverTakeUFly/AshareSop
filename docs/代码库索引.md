# 代码库索引

> 本文件是 `stockhot/` + `davis_analyzer/` 代码模块的总导航索引，按**工具类 / 引擎类 / 数据类 / 研报类**分类组织。
> 与 [README.md](README.md) 互补：README 管 `docs/` 下的研究文档产物，本文件管代码模块。
> **最近更新**：2026-07-17

## 📁 顶层目录总览

```
CodeAgentDashboard/
├── stockhot/              ← 盘面采集 + 交易建议 + 盘后复盘（主业务包）
├── davis_analyzer/        ← 量化选股引擎（戴维斯双击 / 景气度 / 困境反转）
├── dashboard/             ← stockhot 数据看板（Next.js + TS，前端）
├── davis_webui/           ← davis Web 界面（FastAPI + Next.js，前后端）
├── docs/                  ← 研报与文档库（见 README.md）
├── studies/               ← 一次性研究脚本与输出产物
├── scripts/               ← 顶层工具脚本（音频生成 / 隧道管理）
├── storage/               ← 运行时数据（SQLite / 缓存 / 报告文件）
├── docker/                ← Dockerfile + 编排文件
├── public/                ← 早期静态前端（vanilla JS）
└── tests/                 ← 顶层集成测试
```

## ⚙️ 引擎类（Engines / 分析引擎）

计算密集型模块：评分、因子、指标、信号、回测、归因。

### stockhot 引擎

| 路径 | 职责 |
|---|---|
| `stockhot/eod_review/` | **盘后量化复盘引擎**：直连 Tushare 拉日线，板块聚合 / 涨停 5 类归因 / 多维情绪温度计（融资融券+北向+大宗）/ N 日趋势对比。独立于 daily-market-scan |
| `stockhot/advisor/` | 投资顾问引擎：`recommendation_engine` + `signal_aggregator` + 硬编码冲突仲裁，含 ask/daily/watchlist CLI |
| `stockhot/ai_analyzer/` | LLM 驱动分析（热点分析/报告，prompt 模板在 `prompts/templates.py`） |
| `stockhot/hotspot_discovery/` | 热点发现 v2.5：板块 + THS 资金流 + 强势股样本 + 公开证据包 + 政策/新闻输入层 |
| `stockhot/index_technical/` | 指数技术分析编排：OHLCV → 指标 → composite_score → 6 阶段分类（支撑压力位/均线/盘前预期） |
| `stockhot/technical_analyzer/` | 个股技术分析：MA/RSI/MACD/KDJ/布林/量比 六维指标 + 综合评分（0-100） |
| `stockhot/sector_volatility/` | 板块 RV 聚合器：成分股 RV20 等权平均为板块 RV（避免龙头主导） |
| `stockhot/volatility/` | 波动率分析器：Layer1 RV + Layer2 分位数 + Layer5 IV/V-R 比（中国版 VIX 框架） |
| `stockhot/sell_monitor/` | 持仓卖出信号监控：hard_stop / target_reached / thesis_broken 三信号 |
| `stockhot/valuation/` | 相对估值三种方法：相对 PE 溢价率 / ERP 法 / PE-Band 二维定位 |

| `davis_analyzer/backtest.py` | 定期调仓回测引擎（point-in-time 评分 → top-N → 次日执行） |

## 🔗 模块依赖关系速查

```
数据流（采集 → 分析 → 报告）：

  daily-market-scan skill
    ├── limit_up / dragon_tiger / fund_flow / risk_alert  (采集)
    └── → data_layer.storage (stockhot.db)
           ↓
  after-hours-review skill  (读 SQLite + web 搜索 → 盘后总结)
  eod_review engine          (直连 Tushare → 量化复盘)

选股流（财务 → 因子 → 评分）：

  davis_analyzer
    ├── tushare_client / financial_fetcher / stock_universe  (数据)
    ├── valuation / trend / prosperity / distress / momentum  (因子引擎)
    ├── scoring / pipeline  (评分 + 编排)
    └── → report_generator / checklist_generator  (研报)

交易流（信号 → 建议 → 模拟）：

  advisor (信号聚合 + 冲突仲裁 → 建议)
    ← technical_analyzer / sell_monitor / davis_analyzer
    → paper_trading (模拟盘执行)
    → notification (Telegram 推送)
```

## 📌 边界说明

1. **`stockhot/data/` 与顶层 `data/`**：均为空目录且 git 未跟踪，疑似历史遗留占位。

4. **`studies/`**：研究脚本不改引擎源码，作为一次性取数/验证工具，遵循 `research-report/references/engine-usage.md` 模板。
