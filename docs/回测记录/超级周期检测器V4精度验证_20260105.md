# 超级周期检测器 V4 — 利润方向过滤 + 精度验证

> **核心改进**：新增利润增速>0 过滤条件，排除「增收不增利」标的
> **验证方法**：2026-01-05 全A股扫描，检验到 07-10 实际涨幅
> **生成日期**：2026-07-12

---

## 一、V3→V4 效果对比

| 指标 | V3 | V4 | 变化 |
|------|-----|-----|------|
| 命中率 | 12.7%（673只） | **7.2%（384只）** | -43% 噪音被砍掉 |
| 涨>50% 精度 | 22.5% | **22.9%** | +0.4pp（持平） |
| 涨>30% 精度 | 30.4% | **31.9%** | +1.5pp（小幅提升） |
| 涨>0% 精度 | 47.2% | **47.7%** | +0.5pp |

### 关键结论

**利润过滤器的核心价值是减少噪音而非提升精度。** 命中数从 673 降到 384（-43%），
但精度几乎不变。这说明被砍掉的 289 只标的中，涨和跌的比例与整体一致——
它们确实是低质量信号（增收不增利），但碰巧也有一些涨了的。

从 practical 角度：**更少的命中 + 同等的精度 = 更高的信号效率。**
每只命中标的的 expected return 不变，但 false positive 绝对数量减少了 43%。

## 五、下一步

1. **emerging 是实战主信号层**：27.1% 精度（3x 随机），且 whitelist_boost 是
   最稳定的触发模式。confirmed 层精度反而更低（16.4%），不适合单独使用。

2. **补充资产负债表维度**：当前仍只用利润表（营收+利润增速），缺少：
   - 合同负债/预收款增速（需求端先行指标）
   - 资本开支 capex 增速（供给端扩张信号）
   这些维度有望进一步提升 precision

3. **potential 层可安全移除**：V3 的 4 只和 V4 的 2 只全部亏损。
   压制高增长假说在当前数据上不成立。
