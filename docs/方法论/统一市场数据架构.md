# 统一市场数据架构方法论

> **创建日期**：2026-07-15 | **状态**：第一阶段落地完成（1A-1D）
>
> 本文档记录 A 股市场数据采集、存储、访问的统一架构决策，作为所有数据相关开发的规范基准。

## 一、背景：为什么要统一

### 1.1 重构前的三大问题（2026-07-15 诊断）

| 问题 | 现状 | 后果 |
|------|------|------|
| **采集未运行** | `crontab.txt` 写了 17:00 跑 `run_daily_scan.py`，但 crontab 从未安装到系统；老 APScheduler 自 2026-04-29 起已停 | 07-13/14/15 连续三天无盘面数据，盘后总结只能现场拉 Tushare 重建 |
| **三库隔离** | `stockhot.db`（盘面+投研，JSON blob）、`tushare_cache.db`（量化，1.43GB 结构化）、3 个孤儿库并存 | 个股日线被 stockhot(`pro_bar`无缓存) 和 davis(`daily`有缓存) 各拉一遍 |
| **客户端碎片化** | `safe_tushare_call`(无缓存)、`TushareClient`(有缓存)、`get_pro_api`(无缓存)、裸`ts.pro_api`(违规) 四套并存 | 同一 token 4 种读法，macro/valuation/fund_flow 违反自家 data-source-convention 规范 |

### 1.2 重复取数的 6 个热点

| # | 重复数据 | 取数方 A | 取数方 B |
|---|----------|---------|---------|
| 1 | 个股日线 | stockhot `technical_analyzer`（`pro_bar`，**无缓存**） | davis `TushareClient`（`daily`+`adj_factor`，有缓存） |
| 2 | 涨停池 `limit_list_d` | `limit_up` 模块 | `sector_volatility` 模块（重拉不读 DB） |
| 3 | 指数日线 `index_daily` | `index_technical` | `volatility` |
| 4 | 估值 `daily_basic` | davis（有缓存） | stockhot advisor（无缓存） |
| 5 | 股票列表 `stock_basic` | davis（7天TTL） | risk_alert + fund_flow（无缓存裸调） |
| 6 | iVIX/QVIX | volatility | invest_sop/overseas |

## 三、核心设计决策

### 3.1 统一 Tushare 网关（tushare_gateway.py）

融合 4 套客户端的优点：

| 设计点 | 来源 | 实现 |
|--------|------|------|
| 新端点 `api.tushare.pro/dataapi` | stockhot safe_tushare_call | 绕过旧版 waditu.com 超时 |
| 线程安全限频 | stockhot RateLimiter + davis 滑窗 | 双轨：5/s 最小间隔 + 400/min 滑窗 |
| 错误分类 | stockhot safe_tushare_call | 权限错误立即失败，频率限制指数退避（1/2/4s） |
| 分页支持 | **新增**（davis 缺失） | 全市场查询自动 offset 分页（5000/页，上限 20 页） |
| 时区强制 | **新增**（修复 davis 隐患） | Asia/Shanghai |
| 双调用风格 | 融合 | `gw.call(api_name, **params)` + 便捷方法 |

### 3.2 三段式增量缓存（cache.py）

继承 davis tushare_client.py 的判定骨架：

```
1) max_cached_date >= end_date  →  历史不可变，直接返回缓存
2) is_fetched_today              →  T+1 数据，今天查过就不再查
3) 否则                          →  增量拉取 max_date 之后的数据
```

修复 davis 两个隐患：①线程安全（补锁）②时区（Asia/Shanghai）。

### 3.3 个股日线去重（technical_analyzer 改造）

**改造前**：`fetch_ohlcv` → `_fetch_via_tushare` → `ts.pro_bar(adj=qfq)`，**无缓存**，每次全量拉。

**改造后**：三级回退
1. **DAL**（market_data.db 缓存）—— 首选，与 davis 共享
2. **Tushare pro_bar** —— DAL 无数据时
3. **AKShare** —— Tushare 失败时

前复权处理：DAL 存原始 close + adj_factor，读取后本地用 `close * adj_factor / latest_adj` 计算（与 pro_bar 的 qfq 语义一致）。

### 3.4 统一采集流水线 + scan_log

**废弃** `stockhot/scheduler/run_daily_workflow`（APScheduler，自 2026-04-29 停），**以** `run_daily_scan.py`（系统 cron）**为唯一基础**。

`run_daily_scan.py` 每个模块执行后写 `scan_log` 表：
- `status`：success / failed / no_data
- `duration_sec`：耗时
- `error_msg`：失败原因
- `rows_affected`：采集行数

盘后总结可通过 `repo.get_scan_log(date)` 查询当日各模块状态，**解决"不知道哪个模块跑没跑"的问题**。

### 3.5 调度：系统 cron

| 时间 | 脚本 | 数据 |
|------|------|------|
| 17:00 Mon-Fri | `run_daily_scan.py` | 涨停/龙虎榜/资金流/技术面/波动率/风险 |
| 20:00-21:30 | overseas/domestic/supply_chain/futures | 海外/事件/产业链/期货情绪 |
| 08:15 Mon-Fri | `run_daily_advisor.py` | AI 建议 + 盘前报告 |
| 08:30 Mon-Fri | `morning_confirm.py` | 早盘确认 |

安装：`bash stockhot/invest_sop/install.sh`（marker 追加，不覆盖）。

## 五、使用指南

### 5.1 读取个股日线（stockhot + davis 共享缓存）

```python
from stockhot.data_layer import get_repository

repo = get_repository()
df = repo.get_daily_prices("000001.SZ", "20260101", "20260715")
# 列：ts_code, trade_date, open, high, low, close, pre_close, pct_chg, vol, amount, adj_factor
```

### 5.2 读取全市场某日行情

```python
df = repo.get_daily_by_date("20260715")  # 自动分页拉取
```

### 5.3 采集日志查询

```python
log = repo.get_scan_log("2026-07-15")
# [{'module': 'limit_up', 'status': 'success', 'duration': 0.6, 'rows': 72}, ...]
```

### 5.4 technical_analyzer 自动走 DAL

```python
from stockhot.technical_analyzer.data_loader import fetch_ohlcv
df = fetch_ohlcv("600519.SH", "2026-07-01", "2026-07-15", adjust="qfq")
# 自动优先走 market_data.db 缓存，未命中才调 Tushare
```

## 六、维护说明

- **数据库路径**：`stockhot/core/config.py` 的 `STORAGE_DIR` → `storage/database/market_data.db`
- **schema 变更**：`stockhot/data_layer/market_db.py` 的 `_SCHEMA_STATEMENTS`，幂等（IF NOT EXISTS）
- **WAL 模式**：market_data.db 使用 WAL + synchronous=NORMAL，读多写少场景下并发性能优
- **token**：统一从 `.env` 的 `TUSHARE_TOKEN` 读取（stockhot 与 davis 同源）
