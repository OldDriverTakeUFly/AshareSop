```bash

# 2. 策略信号生成（手动或脚本）
```

- HMM 牛熊状态 + 波动率状态 + 仓位建议

## 五、关键教训（8 条）

### 1. IC 噪音 > 信号，必须配合分组回测
单日 Spearman IC 标准差是均值的 14 倍。分组回测才能看清尾部分离。

### 2. 量价的价值在卖出层，不在买入层
`enable_volume_risk` 贡献 +3.66pp，`volume_weight` 越高反而拖累。

### 3. 实证有效 ≠ 集成有效
事件因子在全市场实证显著，但强势股 universe 集成时反向。集成效果必须回测验证。

### 4. 集中持仓 + 紧止损 = 最优 Sharpe
`pos=5 + stop=0.70` 全面优于 `pos=10/12`。分散在熊市引入噪音。

### 5. 波动率是风险调节器，不是方向裁判
高波动 = 宽止损 + 轻仓位。不进入牛熊判断，只调风控参数。

### 6. HMM 比人工规则更客观
5 年数据训练的 3 状态 HMM 比旧的 5d/20d 收益率规则准确得多（Sharpe +0.252 → +1.398）。

### 7. 策略定位：趋势跟踪，不追单日反弹
半导体涨停但 5 日仍跌 → 策略正确地不选入。坚持确定性趋势。

### 8. 数据基础设施决定上限
daily_price 自动更新、指数 5 年历史、事件入库 —— 这些基础工作决定了策略能做多远。
