# Prosperity Audit Checklist (景气度审计清单)

当对个股或行业做景气度投资分析时，在给出周期定位结论前使用此清单。逐轴确认景气度信号完整、ΔG 方向明确、山峰位置如实分类，避免因数据不全或主观偏差导致的误判。

## 六维指标完整性

- [ ] BB Ratio（订单出货比）趋势方向已确认：标注当前值、与历史对比、上行或下行方向
- [ ] 稼动率（产能利用率）水平已确认：标注当前值是否高于 90%（涨价压力）或低于 70%（产能过剩）
- [ ] 交期水平已确认：标注当前交期周数、与正常基线（8-10 周）的比值、是否已超过 16 周或 30 周阈值
- [ ] 经销商库存水平已确认：标注当前库存月数、与正常基线（3-6 月）的对比、是否降至 1-1.5 月紧张区间
- [ ] 排产周期已确认：标注厂商排产延伸月数、是否超过 12 个月或 24 个月
- [ ] LTA（长期协议）扩散情况已确认：标注下游客户签订 LTA 的规模和期限趋势
- [ ] 六维方向一致性已核对：六个维度是否同向上行（最强信号）；是否存在背离（背离本身是警告）

## ΔG 信号验证

- [ ] 财务数据覆盖至少 4 个季度（ΔG 计算的最低要求）
- [ ] ΔG 符号已确认：通过 `davis_analyzer.prosperity.calculate_prosperity_score` 的 `delta_g` 字段获取
- [ ] ΔG 为正时：标注"增速加速中"，对应左山峰/加速期，确认是否为"二次点火"候选
- [ ] ΔG 为负时：标注"增速减速中"，对应右山峰/减速期，确认是否触发 30% 阈值预警
- [ ] ΔG 符号翻转时点已追踪：通过 `prosperity_inflection.identify_inflection_quarter` 确认拐点季度
- [ ] 行业内相对 ΔG 已计算：通过 `prosperity_sector.compute_relative_delta_g` 确认个股相对行业的加速度位置

## 30% 阈值核查

- [ ] 净利润增速当前值已标注，是否高于或低于 30%
- [ ] 若增速低于 30%：确认降幅是否超过 50%、减速斜率是否陡峭（三条件叠加 = 拐点强信号）
- [ ] 若增速高于 100%：确认降幅容忍度可适当放宽至约 60%

## 山峰理论定位

- [ ] 左山峰（Δg > 0）：确认是否为戴维斯双击机会区，标注上行段位置
- [ ] 山顶（G 最高，Δg 趋零）：确认是否为奇点时刻过渡区，标注增速是否即将减速
- [ ] 右山坡（Δg < 0 但 G 仍正）：确认是否为估值收缩风险区，标注减速起点
- [ ] 山后（G 转负）：确认是否为戴维斯双杀区，标注回避标记

## 成长股投资时钟分类

- [ ] 周期阶段已通过 `prosperity_sector.classify_stock_stage` 分类：加速期 / 减速期 / 上升拐点 / 下降拐点
- [ ] 分类结果与山峰理论定位一致（加速期对应左山坡、减速期对应右山坡）
- [ ] 若行业 β 景气但个股 α 分化：已通过相对 ΔG 区分"行业景气 ≠ 个股加速"，不做强行乐观

## 位置感框架验证

- [ ] 位置（位置）：周期阶段已通过上述分类确认
- [ ] 空间（空间）：渗透率位置已评估（<5% 导入期 / 5-15% 成长期早期 / 15-30% 成长期中后期 / >30% 成熟期）
- [ ] 追踪（追踪）：监控指标已设定（月度景气预期斜率、季度 ΔG 更新、六维仪表盘、交期趋势）
- [ ] 坐标（坐标）：横向对比已完成（同行业 peer、跨行业排名、历史周期对比，确认 2.5 年统计锚点是否适用）

## 引擎调用验证

- [ ] 调用 `prosperity.calculate_prosperity_score` 获取复合评分和四维子分
- [ ] 调用 `prosperity_sector.classify_stock_stage` 获取阶段分类
- [ ] 若筛选二次点火标的：调用 `prosperity_sector.screen_g_delta_g_ignition`
- [ ] 调用 `prosperity_inflection.analyze_inflection` 获取拐点分析和叙述
- [ ] 若需行业聚合：调用 `prosperity_sector.aggregate_industry_prosperity`
- [ ] 未修改 `prosperity.py`、`prosperity_inflection.py`、`prosperity_sector.py` 源码（只读调用）

## 结论判定

- [ ] 全部轴通过 → 景气度分析完整，周期定位可信，可输出结论
- [ ] 六维指标不足 4 维 → 标注"数据不完整"，结论降级为"参考性判断"，不可作为核心配置依据
- [ ] ΔG 数据不足 2 个季度 → 标注"ΔG 不可靠"，周期定位降级为"待确认"
- [ ] 山峰定位与时钟分类不一致 → 标注分歧，以引擎分类为准，人工复核数据
