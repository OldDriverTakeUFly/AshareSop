# Report Completeness Checklist

当执行 invest-sop-pre-market 生成盘前报告或晨间指令时，在报告写入磁盘后、向用户汇报结果前使用此清单。逐项确认数据源已查到、报告节齐全、决策边界语言正确、错误已按 §6 处理。

## 数据源确认（盘前报告 Workflow A）

- [ ] **`invest_overseas_market` 已查当日行？** — 若空，§1.1 海外市场须标注「数据不可用」
- [ ] **`invest_domestic_events` 已查当日行？** — 若空，§1.2 须标注「无重大事件记录」
- [ ] **`invest_futures_sentiment` 已查当日行？** — 若空，§1.1 股指期货行须标注「数据不可用」
- [ ] **`invest_cycle_assessments` 已查（最新，无日期过滤）？** — 若空，§2 须提示「请先运行 weekly_cycle.py 更新周期评估」
- [ ] **`invest_holdings` 已查 status='active'？** — 若空，§3 须显示「（无活跃持仓）」
- [ ] **`advisor_runs` 已查当日？** — 即使为空，「AI 综合建议」节也会输出（空时各子表为空）

## 报告节完整性（实际输出 9 节）

- [ ] **§1 市场环境评估** — 4 小节（海外市场 / 重大事件 / 国内政策 / 综合判断）
- [ ] **§2 板块周期评估** — 9 个固定板块行（AI/半导体/软件/锂电/光伏/新能源车/有色/化工/煤炭）
- [ ] **§3 持仓标的操作决策** — 每只活跃持仓一张四维评估占位表
- [ ] **持仓监控（卖出时机）** — `<!-- SELL_SIGNALS_START/END -->` 包裹，来自 `sell_monitor`
- [ ] **AI 综合建议** — `<!-- ADVISOR_SECTION_START/END -->` 包裹，来自 `advisor.report_integration`，分建仓/调仓/清仓/做T 四子表
- [ ] **§4 新增标的备选** — 占位「（暂无备选标的）」
- [ ] **§5 今日重点关注** — 时间事件 / 关键价位 / 情绪阈值三张占位表
- [ ] **§6 风控检查** — 总仓位 / 单票 / 板块集中度 / 持仓数 / 最小止损距离 5 行占位
- [ ] **§7 昨日复盘** — 占位项

## 晨间指令（Workflow B）

- [ ] **读取了前夜盘前报告？** — `generate_directive.py` 读 `{date}_pre_market.md`
- [ ] **`invest_morning_data` 已查当日？** — 若空，按 SKILL.md §6 不生成 directive，记录告警
- [ ] **输出了「早盘数据对比」与「操作指令表」？** — 列：标的 / 昨夜预案 / 早上修正 / 最终操作 / 价格触发条件

## 决策边界语言（CRITICAL）

- [ ] **操作表述为「矩阵结果」而非指令？** — 写「决策矩阵结果：减仓30%」，不写「建议买入」「应该加仓」
- [ ] **没有捏造数据？** — 缺失数值不得估算或插值
- [ ] **没有作为交易指令下达？** — 报告只呈现分析，最终决策由人工
- [ ] **所有数据库查询是只读 SELECT？** — 不允许 INSERT/UPDATE/DELETE

## 错误处理（SKILL.md §6）

- [ ] **NULL 列显示 N/A？** — 单列为空但行存在时显示「N/A」，不显示空白
- [ ] **核心表（holdings / overseas market）全空时停止？** — 不得生成"看起来完整"的报告
- [ ] **数据库文件不存在时停止？** — 报告需要先跑 migrate.py，不自行建库
- [ ] **§3-7 占位表保留为空模板？** — 生成器只灌 §1/§2/持仓监控/AI 建议；§3-7 的分析内容由人工或后续 agent 根据决策矩阵填充，不强行编造

## 结果汇总

- [ ] **报告文件已写入 `stockhot/invest_sop/reports/{date}_pre_market.md`？**
- [ ] **已向用户报告哪些数据可用、哪些标注「数据不可用」？**
- [ ] **未触犯决策边界（未下单、未改库、未捏造）？**

## 判定

- [ ] 所有项目通过 → 报告完成，可交付用户
- [ ] 核心数据源（holdings / overseas market）空 → 按 §6 停止生成，告知用户先跑数据采集
- [ ] 非核心数据源空 → 仍可生成报告，对应节标注「数据不可用」，其余节正常输出
